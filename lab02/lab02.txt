Tarea.-

 - __init__.py : Determina los archivos XML que serán convertidos durante la inicialización. Determina las dependencias del módulo creado

 - __manifest__.py : Contiene la información del módulo como su nombre, resumen, descripcion, autor, sitio web, tags, categoria y version

 - controllers: Contiene los archivos que servirán para definir las rutas del módulo

 - demo: Contiene un script de llenado de data a modo de demostración.

 - models: Para definir los modelos de clase junto con atributos y funciones.

 - security: Contiene los permisos para los modelos y archivos.

 - views: Contiene las vistas y los registros del modelo a ser mostrado. Cada tipo de vista representa un modo de visualización.

Conclusiones.-

 - Odoo ofrece distintos módulos que se complementan entre si

 - Odoo cuenta con una barra de búsqueda que permite buscar módulos y aplicaciones usando distintos tipos de filtros superpuestos.

 - La instalación de un módulo de Odoo es fácil y rápida, requiriendo solo una refresh del sistema.

 - Para la creación de un módulo en Odoo, primero debemos activar el modo desarrollador que se encuentra en la configuración de Odoo.

 - La estructura de un módulo Odoo contiene diferentes carpetas junto con archivos python y xml, que se encargan de proveer función a dicho módulo.

 - Para que aparezca el módulo creado en el panel de aplicaciónes, primero se debe a actualizar la lista de estas.

 - La desinstalación de un módulo en Odoo tambien es intuitiva y rápida
